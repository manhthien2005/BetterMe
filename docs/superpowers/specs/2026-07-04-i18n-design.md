# BetterMe i18n Design

## Goal

Add bilingual UI support for Vietnamese and English, with English as the final fallback. The app should detect the user's language from the route, saved preference, and browser region, then remember explicit user choices.

## Scope

This is one cohesive product change, not separate sub-projects. Locale resolution, routing, settings persistence, copy dictionaries, and UI controls all serve the same user-facing multilingual behavior.

## Supported locales

- `en`: English, default and final fallback.
- `vi`: Vietnamese.

Unsupported languages resolve to `en`.

## Locale resolution order

1. Locale segment in the URL: `/en/...` or `/vi/...`.
2. Saved cookie preference: `betterme_locale`.
3. Saved local settings field: `TrackerSettings.locale`.
4. Browser or request language, using BCP 47 tags such as `vi-VN` or `en-US`.
5. Final fallback: `en`.

## Routing behavior

Primary app routes become locale-prefixed:

- `/en/dashboard`, `/vi/dashboard`
- `/en/tracker`, `/vi/tracker`
- `/en/calendar`, `/vi/calendar`
- `/en/habits`, `/vi/habits`
- `/en/settings`, `/vi/settings`

Legacy unprefixed routes redirect to the resolved locale equivalent. Root `/` redirects to `/<locale>/dashboard`.

## Persistence behavior

When the user picks a language:

- Save `TrackerSettings.locale`.
- Save cookie `betterme_locale`.
- Move the current route to the same page under the selected locale.
- Keep habit/reflection user content unchanged.

Existing local data without `settings.locale` remains valid and migrates to `en` on load.

## Dictionary strategy

Copy lives in focused TypeScript dictionaries:

- `src/i18n/locales/en.ts`
- `src/i18n/locales/vi.ts`

Components consume typed dictionary keys through `useI18n()`. New UI copy must not be hardcoded directly inside components unless it is user-generated data or a stable product name such as `BetterMe`.

## Data localization rules

- Completion status values stay canonical in data: `Good`, `Okay`, `Bad`, `Planned`.
- UI renders localized status labels.
- Default habits are seeded in the first-run locale.
- User-created habit names and reflections are treated as user content and are not machine-translated.
- Motivation messages are localized through locale-specific data.
- Chart titles, descriptions, labels, and "No data" strings are localized.

## Accessibility

- `<html lang>` is updated to the resolved locale.
- Language controls have localized labels.
- Locale switching must remain keyboard accessible.
- E2E tests cover redirects, Vietnamese copy, English fallback, and saved language preference.

## Constraints

- Do not add an i18n dependency for Phase 1; typed local dictionaries are enough for `vi` and `en`.
- Do not translate user-generated content automatically.
- Keep URL locale, cookie preference, and `TrackerSettings.locale` consistent after explicit user selection.
- Keep existing local storage data readable.
- Preserve existing theme/scoring/storage behavior.
