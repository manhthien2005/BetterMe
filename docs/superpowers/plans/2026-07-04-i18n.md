# BetterMe i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Vietnamese/English localization with locale-prefixed routes, browser/default detection, and remembered user language choice.

**Architecture:** Use typed local dictionaries and a small locale utility layer instead of adding an i18n package. Locale routing is handled by middleware and `[locale]` App Router wrappers; runtime UI copy flows through `LocaleProvider`/`useI18n()`. User preference is saved in both `TrackerSettings.locale` and cookie `betterme_locale`.

**Tech Stack:** Next.js App Router, React context, TypeScript, Zod, Vitest, Playwright.

## Global Constraints

- Supported locales are exactly `en` and `vi`.
- Final fallback locale is exactly `en`.
- Cookie key is exactly `betterme_locale`.
- Local storage schema version remains readable for existing data without `settings.locale`.
- Do not add a new i18n dependency.
- Do not translate user-generated habit names or reflections automatically.
- Keep canonical data statuses as `Good`, `Okay`, `Bad`, and `Planned`.
